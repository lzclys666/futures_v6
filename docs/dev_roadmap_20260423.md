| 2.3 | 下单逻辑实现（限价+市价） | P0 | 策略根据信号方向发单，OmsEngine 收到回报 | 6h |
| 2.4 | 持仓同步与状态管理 | P1 | 策略内部维护持仓状态，支持查询当前持仓/盈亏 | 4h |
| 2.5 | Paper Trade 首次成交 | P0 | 模拟账户产生至少 1 笔成交记录 | 4h |

### 里程碑
- **M2.1**（5.7）：策略根据 CSV 信号成功下单（未成交也算）

- **下单逻辑**：优先限价单，超时未成交转市价单（可配置）
- **除零保护**：所有价格计算必须处理除零、None、异常值

### 风险与应对
| 风险 | 概率 | 应对 |

| CTP 模拟盘流动性不足 | 中 | 使用市价单或放宽限价范围 |
| 信号与行情时间戳不同步 | 中 | 增加信号有效期检查，过期信号丢弃 |
| 风控规则过度拦截 | 低 | 先宽松后严格，逐步调参 |

## Phase 3：连接器模块（W4-W5，5.15-5.28）

**目标**：实现宏观层与 VNpy 执行层的 3 个连接器

### 我的任务清单

| # | 任务 | 优先级 | 验收标准 | 预估工时 |
|---|------|--------|----------|----------|
| 3.1 | VNpyBridge 开发 | P0 | VNpy EventEngine → FastAPI `/api/trading/*` 路由，支持查询持仓/资金/订单 | 10h |
| 3.2 | SignalBridge 开发 | P0 | CSV 变更事件 → VNpy EventEngine，替代轮询逻辑 | 8h |
| 3.3 | MacroRiskApp 开发（基础 3 规则） | P1 | 作为 VNpy App 插件运行，实现持仓限额、亏损限额、价格限制 | 8h |
| 3.4 | API 文档更新 | P1 | `/api/trading/*` 端点接入 Swagger，提供请求/响应示例 | 3h |
| 3.5 | 集成测试：Bridge 端到端 | P1 | pytest 覆盖 VNpyBridge + SignalBridge 联合场景 | 5h |

### 里程碑
- **M3.1**（5.21）：VNpyBridge 打通，FastAPI 能查询 VNpy 持仓/资金
- **M3.2**（5.28）：SignalBridge 事件驱动上线，轮询逻辑下线

### 技术要点
- **VNpyBridge**：使用 `asyncio` 桥接 VNpy 同步事件与 FastAPI 异步路由
- **SignalBridge**：`watchdog` 监控 CSV → 自定义事件 → EventEngine 分发
- **MacroRiskApp**：继承 VNpy `BaseApp`，通过 `EventEngine` 监听订单/成交事件
- **线程安全**：所有跨线程数据传递使用 `queue.Queue` 或 `asyncio.Queue`

### 风险与应对
| 风险 | 概率 | 应对 |
|------|------|------|
| EventEngine 事件丢失 | 中 | 增加持久化队列，关键事件落盘 SQLite |
| FastAPI 与 VNpy 线程冲突 | 中 | 使用 `asyncio` 桥接，避免直接跨线程调用 |
| App 插件加载失败 | 低 | 参考 VNpy 官方 App 开发文档，确保 `__init__.py` 导包正确 |

### 13 条风控规则清单

| # | 规则名称 | 类型 | 说明 |
|---|----------|------|------|
| R1 | 单品种最大持仓 | 持仓 | 单个品种持仓不超过阈值 |
| R2 | 单日最大亏损 | 资金 | 当日亏损达到阈值暂停交易 |
| R3 | 价格涨跌停限制 | 价格 | 价格触及涨跌停禁止开仓 |
| R4 | 总持仓市值限制 | 持仓 | 所有品种持仓总市值不超过阈值 |
| R5 | 单品种最大回撤 | 资金 | 单个品种回撤达到阈值减仓 |
| R6 | 连续亏损次数 | 交易 | 连续 N 笔亏损暂停该品种交易 |
| R7 | 波动率限制 | 市场 | ATR 超过阈值禁止开仓 |
| R8 | 保证金占用率 | 资金 | 保证金占用不超过账户权益阈值 |
| R9 | 交易时间限制 | 时间 | 非交易时间禁止下单 |
| R10 | 信号强度过滤 | 信号 | 信号绝对值低于阈值不交易 |
| R11 | 品种相关性限制 | 组合 | 高相关性品种同时持仓限制 |
| R12 | 撤单次数限制 | 交易 | 单日撤单超过阈值暂停 |
| R13 | 宏观熔断 | 宏观 | 所有品种同向极端信号暂停全系统 |

### 技术要点
- **规则配置**：`config/risk_rules.yaml`，支持动态启用/禁用、参数热更新
- **审计日志**：SQLite 表结构 `audit_logs(order_id, event_type, timestamp, details)`
- **熔断恢复**：手动恢复 + 定时自动恢复（可配置）
- **除零保护**：所有风控计算必须处理除零、None、异常值

### 风险与应对
| 风险 | 概率 | 应对 |
|------|------|------|
| 风控规则过度拦截 | 中 | 逐步调参，先宽松后严格，记录每次触发原因 |
| 熔断误触发 | 中 | 增加确认延迟（如信号持续 3 分钟才触发） |
| 审计日志性能瓶颈 | 低 | 批量写入、异步落盘、定期归档 |

### 风险与应对
