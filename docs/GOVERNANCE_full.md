## 一、核心原则

### 1.1 联邦制治理

系统规模已达10+Agent协作，传统中央集权模式已失效。采用**联邦制+强契约**模式：
- 各Agent保持自主决策权
- 核心接口强制锁定（契约高于一切）
- 重大决策集体投票

### 1.2 三不原则

```
不透明 → 透明（所有通信走事件总线）
不记录 → 记录（所有决策入BUG_TRACKER/决策日志）
不审计 → 审计（所有代码变更必须审计Agent通过）
```

### 1.3 惩罚机制

| 违规类型 | 首次 | 二次 | 三次 |
|----------|------|------|------|
| 未走事件总线 | 健康度-10 | 健康度-30 | 停职审查 |
| BUG未登记 | 健康度-15 | 冻结任务 | 停职审查 |
| 接口变更未审批 | 强制回滚 | 健康度-50 | 移除权限 |
| 审计不通过 | 打回重做 | 健康度-20 | 技术议会介入 |
