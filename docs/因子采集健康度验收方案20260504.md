### L1.2 — 批量 `import` 验证（脚本化，10min）

对每个因子脚本，通过 `importlib` 动态导入模块。由于路径依赖 `sys.path.insert`，需先添加搜索路径：

```python
import sys, os, importlib.util

CRAWLERS = "D:/futures_v6/macro_engine/crawlers"
errors = []

for root, dirs, files in os.walk(CRAWLERS):
    # 跳过系统目录
    if "__pycache__" in root or "common" in root or "_shared" in root:
        continue
    for f in files:
        if not f.endswith(".py") or f.startswith("_") or f == "__init__.py":
            continue
        # 模拟脚本的 sys.path.insert
        sys.path.insert(0, os.path.join(CRAWLERS, os.path.basename(root)))
        sys.path.insert(0, CRAWLERS)
        
        mod_name = f.replace(".py", "")
        try:
            importlib.import_module(mod_name)
        except ImportError as e:
            errors.append((f, str(e)))
        
        # 清理 path 避免膨胀
        sys.path.pop(0)
        sys.path.pop(0)

print(f"通过: {total - len(errors)}/{total}, 失败: {len(errors)}")
for f, e in errors[:20]:
    print(f"  ❌ {f}: {e}")
```

**验收标准**: 通过率 **≥ 95%**（允许少量低频因子因网络库缺失而失败）

**常见错误模式**：
```python
# ❌ 错误：没有解包返回值
html = fetch_url(url)  # fetch_url 返回 tuple，html 实际上是 (str, None)

# ❌ 错误：仍把返回值当 response 对象
result = fetch_url(url).text  # tuple 没有 .text 属性

# ✅ 正确
html, err = fetch_url(url)
if err:
    return None
# 然后使用 html
```

## L3：数据库审计（30min，全程脚本化）

**目标**: 检查 pit_data.db 中各因子是否有合理的最新数据，**并强制验证 PIT 合规性和因子覆盖率**

### L3.1 — 最近数据日期检查

```sql
-- 查看每个品种最近一次数据更新时间
SELECT symbol, factor_code, MAX(obs_date) as last_data, COUNT(*) as records
FROM factor_data
GROUP BY symbol
ORDER BY last_data DESC;
```

**验收标准**:
- 核心品种(P0/P1)：最近数据应在 **3 天内**（含非交易日）
- 次核心品种：最近数据应在 **7 天内**
- 如果某品种 last_data 超过 14 天 → **红色告警**（该品种因子链断裂）

### L3.2 — 数据缺失率检查

```sql
-- 查看每个品种的采集覆盖率
SELECT 
    symbol,
    COUNT(DISTINCT factor_code) as active_factors,
    MIN(obs_date) as earliest,
    MAX(obs_date) as latest
FROM factor_data
GROUP BY symbol
ORDER BY active_factors DESC;
```

**验收标准**: 应有 **≥ 240 个** 活跃因子（根据系统架构文档）

### L3.3 — 异常值检查

```sql
-- 检查异常大/小的值
SELECT symbol, factor_code, value, obs_date
FROM factor_data
WHERE 
    (symbol = 'AU' AND (value < 0 OR value > 5000)) OR
    (symbol = 'CU' AND (value < 0 OR value > 100000))
ORDER BY value DESC
LIMIT 20;
```

**验收标准**: 无极端异常值（单位错误、小数点错位）

### L3.4 — PIT 合规性检查（新增，强制）

```sql
-- 检查是否有未来数据（PIT 违规）
SELECT symbol, factor_code, obs_date, value
FROM factor_data
WHERE obs_date > date('now')
ORDER BY obs_date DESC;
```

**验收标准**: **0 条记录**。任何未来数据都是 PIT 违规，必须修复。

### L3.5 — 因子覆盖率检查（新增）

```sql
-- 检查每个品种的有效因子数量
SELECT 
    symbol,
    COUNT(DISTINCT factor_code) as effective_factors
FROM factor_data
WHERE obs_date >= date('now', '-30 days')  -- 30天内有数据的因子
GROUP BY symbol
ORDER BY effective_factors DESC;
```

**验收标准**:
- P0 品种（AU/CU/RB/AG/RU）：有效因子数 **≥ 15**
- P1 品种（J/AL/NI/TA/HC/JM/SA/ZN/P/PB）：有效因子数 **≥ 10**
- 低于上述门槛 → 🔴 告警（因子链断裂或采集停滞）

## 告警规则

验收过程中发现的异常按以下级别分类：

| 告警级别 | 触发条件 | 响应 |
|---------|---------|------|
| 🟡 警告 | 单因子首选抓取失败（但备选成功）/ 数据质量校验失败（值在边界附近） | 记录日志，不阻断验收 |
| 🟠 注意 | 单因子首选+备选都失败（仅兜底保障回补）/ 品种最近数据超过 7 天 | 记录到 ISSUES.md |
| 🔴 严重 | 首选抓取连续失败 3 次 / 三源（首选+备选+兜底）全部失败 / 数据与替代源差异 > 1% / PIT 违规（未来数据） / 因子覆盖率低于 P0≥15 或 P1≥10 | 阻断验收，暂停流程，通知因子分析师 |

### run_all 集成测试中的告警应用

| 场景 | 级别 | 响应 |
|------|------|------|
| 子因子首选抓取成功，写入正常 | 🟢 正常 | 无动作 |
| 子因子首选抓取失败，备选抓取成功 | 🟡 警告 | 记录哪个源失败 + 哪个源成功 |
| 子因子首选+备选都失败，兜底保障回补成功 | 🟠 注意 | 记录到 ISSUES.md，标记为"缺实时源" |
| 子因子三层全部失败（无历史值可回补） | 🔴 严重 | 标记 ⛔永久跳过，通知因子分析师 |
| 同品种 3+ 因子同时 🔴 | 🔴 严重 | 暂停该品种验收，排查系统性故障 |

## 附录：批量验收脚本（更新版）

将以下脚本保存为 `tools/run_verification.py`，可一键执行 L0/L1/L3 的批量验证：

```python
#!/usr/bin/env python3
"""因子采集健康度验收 v2.2 — 批量验证脚本"""
import sys, os, importlib, subprocess, sqlite3, json
from pathlib import Path
from datetime import datetime, timedelta

CRAWLERS = Path("D:/futures_v6/macro_engine/crawlers")
DB_PATH = Path("D:/futures_v6/macro_engine/pit_data.db")

results = {"l0": {}, "l1": {}, "l3": {}}

# === L0: 文件系统检查 ===
print("[L0] 检查 requests.get 残留...")
result = subprocess.run(
    "grep -rl 'requests.get' macro_engine/crawlers/ --include='*.py' 2>/dev/null | grep -v __pycache__ | grep -v common/ | wc -l",
    shell=True, capture_output=True, text=True, cwd=str(CRAWLERS.parent.parent)
)
results["l0"]["requests_get_remaining"] = int(result.stdout.strip())
results["l0"]["pass"] = (results["l0"]["requests_get_remaining"] == 0)

# === L1: 批量 import 验证 ===
print("[L1] 批量导入验证...")
sys.path.insert(0, str(CRAWLERS))
errors = []
total = 0

for root, dirs, files in os.walk(CRAWLERS):
    if "__pycache__" in root or "common" in root or "_shared" in root:
        continue
    for f in files:
        if not f.endswith(".py") or f.startswith("_") or f in ("__init__.py",):
            continue
        total += 1
        sub_dir = os.path.basename(root)
        sys.path.insert(0, os.path.join(CRAWLERS, sub_dir))
        sys.path.insert(0, str(CRAWLERS))
        mod_name = f.replace(".py", "")
        try:
            importlib.import_module(mod_name)
        except ImportError as e:
            errors.append({"file": f, "dir": sub_dir, "error": str(e).split("\n")[0][:80]})
        sys.path.pop(0)
        sys.path.pop(0)

results["l1"]["total"] = total
results["l1"]["passed"] = total - len(errors)
results["l1"]["errors"] = errors[:30]
results["l1"]["pass"] = (len(errors) / total <= 0.05)

# === L3: 数据库审计 ===
print("[L3] 数据库审计...")
conn = sqlite3.connect(str(DB_PATH))
cursor = conn.cursor()

# 最近数据日期
cursor.execute("""
    SELECT symbol, MAX(obs_date) as last_data, COUNT(DISTINCT factor_code) as factors
    FROM factor_data GROUP BY symbol ORDER BY last_data DESC
""")
db_summary = cursor.fetchall()
results["l3"]["total_symbols"] = len(db_summary)
results["l3"]["recent_3d"] = sum(
    1 for s, d, _ in db_summary 
    if d and (datetime.strptime(d, "%Y-%m-%d").date() >= (datetime.now() - timedelta(days=3)).date())
)

# PIT 合规检查（新增）
cursor.execute("""
    SELECT COUNT(*) FROM factor_data WHERE obs_date > date('now')
""")
pit_violations = cursor.fetchone()[0]
results["l3"]["pit_violations"] = pit_violations
results["l3"]["pit_pass"] = (pit_violations == 0)

# 因子覆盖率检查（新增）
cursor.execute("""
    SELECT symbol, COUNT(DISTINCT factor_code) as effective_factors
    FROM factor_data
    WHERE obs_date >= date('now', '-30 days')
    GROUP BY symbol
""")
coverage = cursor.fetchall()
p0_symbols = {'AU', 'CU', 'RB', 'AG', 'RU'}
p1_symbols = {'J', 'AL', 'NI', 'TA', 'HC', 'JM', 'SA', 'ZN', 'P', 'PB'}
coverage_dict = dict(coverage)

p0_adequate = sum(1 for s in p0_symbols if coverage_dict.get(s, 0) >= 15)
p1_adequate = sum(1 for s in p1_symbols if coverage_dict.get(s, 0) >= 10)
results["l3"]["p0_coverage_adequate"] = p0_adequate
results["l3"]["p1_coverage_adequate"] = p1_adequate

# 异常值告警
cursor.execute("""
    SELECT symbol, factor_code, value, obs_date FROM factor_data
    WHERE (symbol = 'AU' AND (value < 0 OR value > 5000))
       OR (symbol = 'CU' AND (value < 0 OR value > 100000))
    ORDER BY value DESC LIMIT 20
""")
results["l3"]["anomalies"] = cursor.fetchall()
conn.close()

# === ic_heatmap 覆盖检查（新增）===
print("[L5.0] 检查 ic_heatmap 品种覆盖...")
try:
    conn = sqlite3.connect(str(DB_PATH))
    cur = conn.cursor()
    cur.execute("SELECT COUNT(DISTINCT symbol) FROM ic_heatmap")
    ic_count = cur.fetchone()[0]
    results["l5"]["ic_heatmap_symbols"] = ic_count
    results["l5"]["ic_heatmap_pass"] = (ic_count >= 20)
    conn.close()
except Exception as e:
    results["l5"] = {"error": str(e)}

# === 输出 JSON ===
print(json.dumps(results, indent=2, default=str))
```
