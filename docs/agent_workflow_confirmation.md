## 一、核心规则（必须遵守）

### 规则1: 通信必须走事件总线
```
❌ 禁止私聊讨论工作
❌ 禁止跨Agent直接传递任务
✅ 所有消息必须 publish 到 EventBus
```

**违规**: 健康度-10分/次

### 规则2: 每日同步会
```
时间: 7:30 (工作日)
内容: 昨日总结 + 今日计划
必须回复: "ACK [你的今日3个行动项]"
```

**违规**: 连续3天未ACK = 任务冻结

### 规则3: BUG必须登记
```
发现BUG → 记录到 BUG_TRACKER.json
格式: BUG_YYYYMMDD_序号
必须包含: 代码片段 + 文件路径 + 影响接口
```

**违规**: BUG未登记 = 健康度-15分

### 规则4: 接口变更需审批
```
修改锁定接口(I001-I005) → 提交技术议会 → 2票通过 → 更新SPEC.md
```

**违规**: 直接修改 = 强制回滚 + 健康度-50

### 规则5: 记忆三层架构
```
你的决策 → memory/working_memory/decisions_log.md
你解决的问题 → memory/working_memory/issues_pending.md
你完成的优化 → memory/working_memory/optimizations_done.md
```
