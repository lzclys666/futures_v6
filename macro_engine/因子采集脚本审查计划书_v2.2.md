## 📋 完整审查清单

### A. 脚本可运行性（3项）

```
- [ ] 脚本是否可通过 python xxx.py --auto 运行（测试最新交易日）
- [ ] 是否有直接 import requests（应通过 AKShare 或 web_utils）
- [ ] subprocess 模式是否设置 PYTHONIOENCODING=utf-8
```

### B. 数据质量（4项）

```
- [ ] 数据是否写入 pit_data.db（查询验证）
- [ ] raw_value 是否为数值类型（int/float，禁止 None/"nan"/字符串）
- [ ] obs_date 是否正确（周一是否回退到上周五）
- [ ] 数据是否在合理范围内（bounds 检查）
```

### C. 异常处理（3项）

```
- [ ] 是否有裸 except:（应指定异常类型 + logging）
- [ ] 网络请求是否有超时设置（requests.get(url, timeout=10)）
- [ ] 是否有重试机制（网络波动时）
```

### D. 编码安全（2项）

```
- [ ] 文件是否含 emoji（✅⚠❌ → [OK][WARN][ERR]）
- [ ] subprocess 是否捕获 stdout/stderr（DEVNULL 或 capture_output）
```

### E. 因子注册（3项）

```
- [ ] 因子是否在 config/factors/{SYMBOL}/ 中有 YAML 定义
- [ ] L4 因子是否有初始值记录（日期 + 来源）
- [ ] 永久跳过因子是否在 run_all.py 中注释掉（不执行）
```

### F. 多数据源漏斗（5项）

```
- [ ] 是否按 L1→L2→L3→L4 顺序独立 try-except（上一层成功即停止）
- [ ] 每层是否打印日志说明用的是哪个源 [L1]/[L2]/[L3]/[L4]
- [ ] source_confidence 是否与实际层数匹配（L1=1.0, L2=0.9, L3=0.8, L4=0.5）
- [ ] L4 回补时是否记录来源为 "db_回补" 或 "L4_historical_fallback"
- [ ] 是否有"跳层"行为（如应先试 AKShare 但直接用 requests 绕过）
```

**漏斗层级定义**：

| 层级 | source_confidence | source 标注示例 |
|------|------------------|----------------|
| L1 免费权威 | 1.0 | "akshare", "FRED", "SGE" |
| L2 免费聚合 | 0.9 | "eastmoney", "sina" |
| L3 付费爬虫 | 0.8 | "mysteel", "SMM" |
| L4 兜底 | 0.5 | "db_回补", "L4_fallback" |

### G. 代码格式（5项）

```
- [ ] 头部是否有完整文档注释（含因子代码/公式/状态/失败原因）
- [ ] 变量命名是否有意义（禁止 a/b/temp/c）
- [ ] 是否有 shebang 行（#!/usr/bin/env python3）
- [ ] sys.path.insert 是否使用相对路径（禁止硬编码绝对路径）
- [ ] 是否符合 PEP 8（缩进4空格，无过长行）
```

**头部文档格式**（必须包含）：

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
SYMBOL_因子名.py
因子: FACTOR_CODE = 中文描述

公式: 计算公式（如果有）

当前状态: [✅正常 | ⚠️待修复 | ⛔永久跳过]
- 失败原因或跳过原因
- 尝试过的数据源及结果
- 解决方案

订阅优先级: ★~★★★★★（付费源才需要标注）
替代付费源: 具体平台名称
"""
```

### H. 输出格式（5项）⭐⭐⭐⭐⭐

```
- [ ] 输出是否使用 [L1]/[L2]/[OK]/[WARN]/[ERR] 等 ASCII 标签（无 emoji）
- [ ] JSON 输出字段是否完整：factor_code, raw_value, obs_date, pub_date, source, source_confidence
- [ ] raw_value 是否为合法数值（禁止 "nan", None, 字符串数值）
- [ ] obs_date/pub_date 是否为 YYYY-MM-DD 格式
- [ ] source 是否标注具体来源平台（如 "akshare_SGE" 而非 "akshare"）
```

**输出格式标准**（stdout 示例）：

```python
# ✅ 正确输出
print(f"[L1] gold={gold_price} CNY/g, ratio={raw_value}")
print(f"[OK] {FACTOR_CODE}={raw_value} obs={obs_date}")
print(f"[WARN] {FACTOR_CODE}={raw_value} 超出bounds{BOUNDS}，跳过")
print(f"[L4] 兜底: {val}")

# ❌ 错误输出（含 emoji 或 raw_value 为非数值）
print("✅ 数据写入成功")  # emoji
print(f"[OK] factor={raw_value}")  # raw_value 可能为 None/"nan"
```

## 🚨 常见问题模式（快速定位）

### F 维度典型问题

```python
# ❌ 错误：跳层（直接用 requests 而非 AKShare）
import requests
data = requests.get(url).json()  # 绕过 L1 AKShare

# ✅ 正确：先用 AKShare，失败后再尝试其他源
try:
    data = ak.xxx()  # L1
except:
    data = fetch_url(url)  # L2
```

### H 维度典型问题

```python
# ❌ 错误：raw_value 为 None 或 "nan"
save_to_db(FACTOR, ..., raw_value=None)       # None
save_to_db(FACTOR, ..., raw_value=float('nan'))  # nan string

# ✅ 正确：bounds 检查后才写入
if BOUNDS[0] <= raw_value <= BOUNDS[1]:
    save_to_db(FACTOR, ..., raw_value=raw_value)
else:
    print(f"[WARN] {FACTOR}={raw_value} 超出bounds，跳过")
```

### I 维度典型问题

```python
# ❌ 错误：参数顺序错误
save_to_db(FACTOR, pub_date, obs_date, raw_value, ...)  # 缺少 symbol

# ❌ 错误：source_confidence 未设置
save_to_db(FACTOR, SYMBOL, pub_date, obs_date, raw_value, source="akshare")
#                                                            # 缺少 source_confidence

# ✅ 正确
save_to_db(FACTOR, SYMBOL, pub_date, obs_date, raw_value,
           source="akshare", source_confidence=1.0)
```
