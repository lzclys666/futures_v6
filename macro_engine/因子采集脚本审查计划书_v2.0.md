**发现**: 约35处裸 `except:`，集中在 run_all.py 和部分采集脚本

**典型模式**:

**涉及文件**:
- AO/EC/LC/LH/NI/P/SC/SN/TA/ZN 等品种的 `_run_all.py`（每文件2处）
- AU_SPDR黄金ETF持仓量.py

- JM_计算焦煤月差.py、JM_计算焦煤期现基差.py
- TA_抓取BRENT价格.py

**严重度**: 🔴 高（异常被吞掉，数据失败无法诊断）

## 📋 审查检查清单（实战版）

### 每品种必查5项

```
#### A. 脚本可运行性（⭐⭐⭐⭐⭐ 最优先）
- [ ] 脚本是否可以通过 `python xxx.py --auto` 运行（测试最新交易日）
- [ ] 是否有 `import requests`（应通过 AKShare 或 web_utils）
- [ ] subprocess 模式是否设置 PYTHONIOENCODING=utf-8

#### B. 数据质量（⭐⭐⭐⭐⭐）
- [ ] 数据是否写入 pit_data.db（查询验证）
- [ ] raw_value 是否为数值（禁止 None/字符串）
- [ ] obs_date 是否正确（周一是否回退到上周五）
- [ ] 数据是否在合理范围内（bounds 检查）

#### C. 异常处理（⭐⭐⭐⭐）
- [ ] 是否有裸 `except:`（应指定异常类型 + logging）
- [ ] 是否有超时设置（requests.get(url, timeout=10)）
- [ ] 是否有重试机制（网络波动时）

#### D. 编码安全（⭐⭐⭐⭐）
- [ ] 文件是否含 emoji（✅⚠❌→ [OK][WARN][ERR]）
- [ ] subprocess 是否捕获 stdout/stderr（防止 tqdm 污染）

#### E. 因子注册（⭐⭐⭐）
- [ ] 因子是否在 `cross_symbol_factors.yaml` 中注册
- [ ] L4 因子是否有初始值记录（日期+来源）
- [ ] 永久跳过因子是否有明确标注（不写占位符）
```

```

   原因: 现有架构是 run_all.py 调度 child script，不存在此函数
```

## 🚨 关键风险提示

### 1. L4因子数据真实性无法自证

AU的4个L4因子（SPDR持仓量/SHFE净持仓/DXY/FED_DOT）写入的是历史回补值或固定stub值，**审查时不应重复运行测试**，否则会用当前不可达的数据源再次写入，导致数据源与实际观测日期不一致。

**建议**: 审查L4因子时，只检查：
- `source_confidence = 0.5`（是否为L4标记）
- `pub_date` vs `obs_date` 是否合理
- **不运行** `AU_run_all.py`，因为L4因子的AKShare数据源当前不可达

### 2. AKShare 接口变更风险

AKShare 经常更新导致接口签名变化（如 `futures_shfe_warehouse_receipt` 之前返回 JSON，之后返回 DataFrame 再到报错）。审查时应**实际运行测试**，不要只看代码静态分析。

**建议**: 每个品种至少运行一次 `XXX_run_all.py --auto`，观察是否有报错。

### 3. 永久跳过因子的状态标签

部分因子已在 ISSUES.md 中标记为"永久跳过"，但脚本头部的状态标签可能未更新。审查时需要对照 ISSUES.md 确认每个永久跳过因子的脚本状态标签为 `⛔永久跳过`。
