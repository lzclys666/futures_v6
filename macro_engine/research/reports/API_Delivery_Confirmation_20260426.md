## 1. IC 热力图 API

### 端点
`GET /api/ic/heatmap`

### 请求参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| symbols | string[] | 否 | 品种列表，默认 ['RB','HC','I','J'] |
| lookback | int | 否 | 回看天数，默认 60 |

### 响应格式
```json
{
  "factors": ["库存变化", "基差率", "现货升贴水", ...],
  "symbols": ["RB", "HC", "I", "J"],
  "icMatrix": [[0.15, -0.08, 0.22, ...], ...],
  "lookbackPeriod": 60,
  "updatedAt": "2026-04-26T15:30:00Z"
}
```

### ✅ YIYI 确认

| 项目 | 确认内容 |
|------|----------|
| IC 计算方法 | **Spearman**（非参数，对异常值稳健） |
| 是否支持按品种筛选 | **是**，symbols参数支持多选 |
| 数据更新频率 | **日终**（收盘后计算） |
| 预计完成日期 | **2026年05月05日** |

## 2. 信号系统 API

### 方案选择
**✅ 选择方案 B：新建端点**

理由：
- 宏观信号与因子信号逻辑不同，分离更清晰
- 避免破坏现有/macro/signal端点的兼容性
- 便于独立迭代和扩展

### 端点
`GET /api/signal/{symbol}`

### 响应格式
```json
{
  "symbol": "RB",
  "compositeScore": 65.5,
  "signalStrength": "BUY",
  "confidence": 78.5,
  "factorBreakdown": [
    {
      "factorName": "库存变化",
      "weight": 0.25,
      "ic": 0.15,
      "contribution": 12.5
    }
  ],
  "regime": "TRENDING",
  "timestamp": "2026-04-26T15:30:00Z"
}
```

### ✅ YIYI 确认

| 项目 | 确认内容 |
|------|----------|
| 方案选择 | **方案 B**（新建端点） |
| 信号聚合方法 | **IC加权求和**（权重=IR归一化） |
| 计算频率 | **日终**（收盘后批量计算） |
| 预计完成日期 | **2026年05月12日** |

### 信号强度定义
| 分值范围 | 信号强度 | 说明 |
|----------|----------|------|
| 0-20 | STRONG_SELL | 强烈看空 |
| 20-40 | SELL | 看空 |
| 40-60 | NEUTRAL | 中性 |
| 60-80 | BUY | 看多 |
| 80-100 | STRONG_BUY | 强烈看多 |

### Regime 定义
| 状态 | 说明 |
|------|------|
| TRENDING | 趋势市场 |
| RANGING | 震荡市场 |
| HIGH_VOLATILITY | 高波动 |
| LOW_LIQUIDITY | 低流动性 |
