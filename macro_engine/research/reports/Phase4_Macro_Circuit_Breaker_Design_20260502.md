## 一、背景与目标

### 1.1 为什么需要宏观熔断

在量化交易系统中，宏观因子（如USD/CNY、CN10Y、WTI原油）理论上应该驱动跨品种的方向性信号。然而，当宏观环境出现极端事件时（如央行突袭式加息、地缘政治危机、疫情封控），所有品种可能同时给出同向极端信号，导致：

- **系统性风险暴露**：组合风险集中爆发
- **因子失效**：IC在极端事件期间可能反转（如COVID-19期间CN10Y→CU关系被政策打破）
- **连环止损**：多个品种同时触及止损，流动性枯竭

宏观熔断机制的目标是：**当系统检测到宏观信号异常时，自动暂停交易并等待人工确认**，防止极端行情下的系统性亏损。

### 1.2 设计目标

| 目标 | 指标 |
|------|------|
| 识别宏观极端信号 | 所有品种信号同向概率 > 80% |
| 识别因子失效区间 | 滚动IC在10个交易日内下降 > 50% |
| 暂停交易 | 熔断触发后立即停止开仓，仅平仓 |
| 人工确认 | 人工确认后恢复交易或延长暂停 |
| 不干扰正常交易 | 假阳性率 < 5%（每月误触发 ≤ 1次） |

## 二、熔断触发条件

采用**三层触发 + 一层确认**机制。

### 2.1 第一层：信号同向度检测（Signal Alignment Check）

**触发逻辑**：
```
当日所有品种的信号方向进行统计：
- long_count = 信号方向为做多的品种数
- short_count = 信号方向为做空的品种数
- total = long_count + short_count（排除 neutral 信号）

alignment_score = max(long_count, short_count) / total

IF alignment_score > 0.80 AND total >= 5:
    → 触发宏观极端告警
```

**为什么 0.80？**
- 正常市场：品种间信号分散，alignment_score ≈ 0.5~0.6
- 极端市场（如2020 COVID）：alignment_score 可达 0.85+
- 阈值 0.80 兼顾灵敏度和稳定性

**信号方向判定**：
```
信号方向 = sign(SignalScore - 50)
- SignalScore > 52 → long
- SignalScore < 48 → short
- 48 ≤ SignalScore ≤ 52 → neutral（不参与统计）
```

### 2.2 第二层：IC衰减检测（IC Decay Check）

**触发逻辑**：
```
对每个品种的核心因子（如 AU→金银比, CU→LME升贴水）计算滚动IC：

IC_decay_ratio = IC_rolling_10d / IC_rolling_60d

IF IC_decay_ratio < 0.50 AND IC_rolling_60d > 0.05:
    → 该品种因子可能进入失效区间
    → 记录警告（不直接触发熔断）

IF 超过 50% 的品种同时触发 IC_decay_warning:
    → 触发宏观熔断（假设宏观逻辑整体失效）
```

**为什么用 IC_rolling_10d / IC_rolling_60d？**
- 比值急剧下降（从0.8降到0.3）= 近期因子方向反转或失效
- 但需要 IC_rolling_60d > 0.05，避免在本来就无效的因子上误触发

### 2.3 第三层：波动率异常检测（Volatility Spike Check）

**触发逻辑**：
```
VIX_index = 系统波动率代理（如使用 WTI 原油波动率或 CN10Y 波动率变化）

IF VIX_index 在 3 个交易日内上升超过 100%:
    → 触发宏观危机告警
```

**数据来源**：
- 主用：WTI原油波动率（akshare）
- 备用：CN10Y收益率变化率
- 兜底：不做波动率检测，仅用前两层

### 2.4 综合触发规则

```
熔断触发 = (第一层 OR 第二层 OR 第三层) AND 人工确认未通过

说明：
- 任一层触发 → 进入"警告"状态
- 警告状态持续 2 个交易日 → 自动升级为"熔断"状态
- 人工可提前触发熔断（手动熔断）
- 人工可撤销警告（人工确认"继续交易"）
```

## 三、熔断动作与恢复

### 3.1 熔断级别

| 级别 | 触发条件 | 动作 |
|------|---------|------|
| **黄色警告** | 任一检测层触发 | 记录日志 + 通知人工；不禁止交易 |
| **橙色警告** | 警告持续2个交易日 | 停止开仓（仅允许平仓）；通知人工确认 |
| **红色熔断** | 橙色警告 + 人工确认失败 OR 手动触发 | 停止所有交易；所有仓位进入对冲模式 |

### 3.2 熔断动作

**橙色级别动作**：
```
1. 停止接受新的开仓信号
2. 对现有持仓发出预警（margin call 风险提示）
3. 通知交易员（邮件/推送）
4. 等待人工确认（30分钟超时自动进入红色熔断）
```

**红色级别动作**：
```
1. 停止所有新单
2. 对冲敞口（如有期权，可买入看跌保护）
3. 通知所有相关方
4. 进入"熔断复盘模式"——等待人工复盘后恢复
```

### 3.3 恢复流程

```
熔断 → 人工复盘 → 判断原因 → 决策：
- 恢复正常交易
- 延长熔断（12h/24h/48h）
- 调整参数后恢复
```

## 五、参数配置

```yaml
circuit_breaker:
  # 第一层：信号同向度
  alignment_threshold: 0.80      # >80% 同向触发
  min_total_signals: 5          # 至少5个品种有信号

  # 第二层：IC衰减
  ic_decay_threshold: 0.50      # 10d/60d < 50% 触发
  ic_min_threshold: 0.05        # 原始IC必须 > 5%
  decay_warning_count: 0.50     # 50%品种触发才熔断

  # 第三层：波动率
  volatility_spike_threshold: 1.0  # 3日上升 > 100%

  # 时间参数
  warning_duration_hours: 2    # 警告持续2个交易日才升级
  manual_confirm_timeout_min: 30  # 人工确认超时30分钟
```

## 六、注意事项与风险

### 6.1 第一期增加人工确认环节

当前设计已包含人工确认（橙色级别超时后才升级红色）。第一期（2026-05-29~06-11）目标：
- 熔断机制运行，但每次橙色触发都必须人工点"确认继续"
- 积累1~2周数据后，评估是否需要调整阈值

### 6.2 假阳性风险

**可能误触发场景**：
- 月末/季末调仓期（大量机构同向调仓）
- 期货合约到期日（移仓导致虚假信号）
- 春节/国庆假期后开盘（跳空高开/低开）

**缓解措施**：
- alignment_threshold 设为 0.80（而非更低）
- 加入"连续触发"条件：需连续2天触发才升级
- 节假日排除：CME/国内期货有明确休市日，休市前后1天不做检测

### 6.3 与现有风控的关系

- 宏观熔断是**额外层**，不是替代现有止损/仓位风控
- 熔断触发后，现有个股/品种级别的止损仍然有效
- 熔断仅影响"是否允许开新仓位"
