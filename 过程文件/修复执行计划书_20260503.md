#### 步骤 2.1 — 建立统一入口（0.5h）

```python
# services/vnpy_bridge.py 尾部添加
_bridge_instance = None
_bridge_lock = threading.Lock()

def set_bridge(instance: "VNpyBridge") -> None:
    global _bridge_instance
    with _bridge_lock:
        _bridge_instance = instance

def get_bridge() -> "VNpyBridge":
    with _bridge_lock:
        if _bridge_instance is None:
            raise RuntimeError("VNpyBridge not initialized. Call set_bridge() first.")
        return _bridge_instance
```

**验证**: 启动 `run.py` 后 `from services.vnpy_bridge import get_bridge; b = get_bridge()` 返回非 None

**具体改造方案 for vnpy.py**:
```python

# 保留: from services.vnpy_bridge import VNpyBridge  (仅用于类型注释)
# 在每个路由函数内:
from services.vnpy_bridge import get_bridge
bridge = get_bridge()
```

**⚠️ 关键风险**: `trading.py` 的 `OrderRequest` Pydantic 模型不依赖 bridge，但路由 handler 在 2.7 行引用了 `bridge` 的 `send_order`、`cancel_order` 等方法。如果 `get_bridge()` 在 bridge 未初始化时被调用，会抛 `RuntimeError`。

**安全策略**: 在 `get_bridge()` 中当 `_bridge_instance is None` 时，返回一个 **`NullBridge` 存根对象**（所有方法返回默认值），而不是抛异常。这样启动时序不对时程序不会崩溃，只是功能降级。

```python
class NullBridge:
    """bridge 未初始化时的安全降级（带日志告警）"""
    _warned_methods = set()  # 避免同一方法重复告警

    def __getattr__(self, name):
        import logging
        if name not in self._warned_methods:
            self._warned_methods.add(name)
            logging.error(
                f"[NullBridge 严重告警] 调用了未初始化的方法: {name}。"
                f"这意味着 bridge 未初始化成功，但系统仍在尝试下单/查询。"
                f"请立即检查 VNpyBridge 启动日志！"
            )
        return lambda *a, **kw: None  # 所有方法返回 None(不抛异常)
    
def get_bridge() -> "VNpyBridge":
    with _bridge_lock:
        return _bridge_instance if _bridge_instance is not None else NullBridge()
```

> **关键区分**：NullBridge 是在**阶段二改造期间**临时使用的兜底机制，不应进入生产环境。  
> **阶段二最终验收标准**：启动 run.py 后，调用 `get_bridge()` 返回真实的 VNpyBridge 实例而非 NullBridge。  
> 验收脚本：`python -c "from services.vnpy_bridge import get_bridge; b = get_bridge(); assert type(b).__name__ != 'NullBridge', 'bridge 未初始化！'"`

**验证**: 完整的启动 + stop + restart 循环，确认每个阶段 API 可用

#### 步骤 4.4 — B3-6.2 web_utils 复用（**精确分步方案**）

**背景**: 24 个脚本 `requests.get()` ≠ 简单的 find & replace。`fetch_url()` 不支持 `headers` 参数，而多达 **14 处**需要自定义 headers（`Referer`、`X-App-Id`、自定义 `Accept` 等），每个网站的请求特征各不相同，批量替换会破坏数据源。其中 2 处的 headers 定义在函数体内（`AU_DXY` 和 `AU_SPDR` 的函数顶部），首次 grep 被遗漏。

#### 4.4.0 — 前置基础：增强 web_utils 接口（0.5h）

```python
def fetch_url(url, encoding=None, timeout=15, retries=2, params=None, headers=None):
    ...
    # 在 _SESSION.get() 处合并 headers：
    req_headers = _SESSION.headers.copy()
    if headers:
        req_headers.update(headers)
    r = _SESSION.get(url, params=params, timeout=timeout, headers=req_headers)
```

```python
# fetch_json 同理
def fetch_json(url, timeout=15, retries=2, params=None, headers=None):
    ...
```
