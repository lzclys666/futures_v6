## 三、逐条规则详解（修订版）

### R1：单品种最大持仓（★动态化修订）

**核心变化：** 从固定比例改为动态计算，考虑品种波动率和相关性聚类

#### 3.1.1 基础公式

```python
def max_position_per_symbol(nav, symbol, all_positions, volatility_20d):
    """
    动态单品种持仓限制
    
    参数:
        nav: 账户净值
        symbol: 品种代码
        all_positions: 当前所有持仓
        volatility_20d: 20日年化波动率
    
    返回:
        max_margin: 该品种最大保证金占用
    """
    # Step 1: 基于波动率调整基础比例
    if volatility_20d > 0.30:      # 高波动（如橡胶、焦煤）
        base_ratio = 0.15
    elif volatility_20d > 0.20:    # 中波动（如铜、原油）
        base_ratio = 0.20
    elif volatility_20d > 0.10:    # 低波动（如黄金、国债）
        base_ratio = 0.25
    else:                           # 极低波动
        base_ratio = 0.30
    
    # Step 2: 考虑相关性聚类
    cluster = get_symbol_cluster(symbol)
    cluster_exposure = calculate_cluster_exposure(all_positions, cluster)
    cluster_limit = nav * 0.40      # 聚类总上限40%
    
    remaining_for_cluster = cluster_limit - cluster_exposure
    
    # Step 3: 综合计算
    symbol_limit = nav * base_ratio
    effective_limit = min(symbol_limit, remaining_for_cluster)
    
    # Step 4: 受总保证金限制
    total_margin = calculate_total_margin(all_positions)
    total_limit = nav * 0.70        # 总保证金70%（预留30%缓冲）
    remaining_total = total_limit - total_margin
    
    return min(effective_limit, remaining_total)
```

#### 3.1.3 计算示例

```
账户净值：100万元
交易品种：螺纹钢（RB）
20日波动率：25%（高波动）
当前持仓：铁矿石（I）多单，占用保证金15万

Step 1: 波动率调整
→ RB波动率25% > 20%，基础比例 = 20%
→ 单品种上限 = 100万 × 20% = 20万

Step 2: 聚类检查
→ RB和I同属"黑色系"
→ 黑色系当前占用 = 15万（I）
→ 聚类上限 = 100万 × 40% = 40万
→ 剩余额度 = 40万 - 15万 = 25万

Step 3: 综合
→ min(20万, 25万) = 20万

Step 4: 总保证金检查
→ 当前总保证金 = 15万
→ 总上限 = 100万 × 70% = 70万
→ 剩余 = 55万

最终：RB最大保证金占用 = min(20万, 55万) = 20万
```

### R2：单日最大亏损（★动态化修订）

**核心变化：** 从固定2%改为基于20日波动率的动态阈值

#### 3.2.1 动态阈值公式

```python
def dynamic_daily_loss_limit(nav, volatility_20d, consecutive_triggers):
    """
    动态单日亏损限制
    
    理论依据:
        - 正常市场：2倍标准差覆盖95%情况
        - 高波动市场：需更严格限制
        - 连续触发后：惩罚递增
    
    参数:
        nav: 账户净值
        volatility_20d: 20日年化波动率
        consecutive_triggers: 连续触发次数（本周）
    """
    # 基础阈值基于波动率
    if volatility_20d > 0.30:      # 极端波动
        base_limit = nav * 0.015    # 1.5%
    elif volatility_20d > 0.20:    # 高波动
        base_limit = nav * 0.020    # 2.0%
    elif volatility_20d > 0.10:    # 中波动
        base_limit = nav * 0.025    # 2.5%
    else:                           # 低波动
        base_limit = nav * 0.030    # 3.0%
    
    # 连续触发惩罚（冷却期递增）
    penalty_factor = 1.0 + (consecutive_triggers * 0.2)
    # 第1次: 1.0, 第2次: 1.2, 第3次: 1.4...
    
    effective_limit = base_limit / penalty_factor
    
    # 绝对下限保护
    return max(effective_limit, nav * 0.005)  # 不低于0.5%
```

#### 3.2.2 计算示例

```
账户净值：100万元
20日波动率：18%（中波动）
本周已连续触发2次

基础阈值 = 100万 × 2.5% = 2.5万
惩罚因子 = 1.0 + (2 × 0.2) = 1.4
有效阈值 = 2.5万 / 1.4 = 1.79万

对比固定阈值：2万 → 动态阈值：1.79万（更严格）
```

**触发后果：** 
- 首次：提示"今日亏损已达动态限制，明日再交易"
- 重复：提示"本周第N次触发，限制已收紧至X%"

### R4：总保证金占用上限（★隔夜缓冲修订）

**核心变化：** 从固定80%改为区分交易时段和隔夜时段

#### 3.4.1 分时段保证金管理

```python
def total_margin_limit(nav, time_of_day, volatility_20d, has_overnight):
    """
    动态总保证金限制
    
    逻辑:
        - 交易时段：可较高杠杆（70%）
        - 收盘前1小时：降低杠杆（60%），预留隔夜缓冲
        - 隔夜持仓：额外预留波动缓冲
    """
    current_time = time_of_day
    
    # 基础限制
    if volatility_20d > 0.25:
        base_limit = 0.60           # 高波动，更保守
    elif volatility_20d > 0.15:
        base_limit = 0.70           # 中波动
    else:
        base_limit = 0.75           # 低波动
    
    # 时段调整
    if is_last_hour_before_close(current_time):
        # 收盘前1小时，降低杠杆
        base_limit *= 0.85
    
    # 隔夜缓冲
    if has_overnight:
        # 预留2倍标准差缓冲
        overnight_buffer = 1 - (2 * volatility_20d / 252**0.5)
        base_limit *= overnight_buffer
    
    return nav * base_limit
```

#### 3.4.2 隔夜风险计算

```python
def overnight_risk_buffer(symbol, volatility_20d, gap_history):
    """
    隔夜跳空风险缓冲
    
    基于历史跳空幅度和当前波动率计算
    """
    # 历史平均跳空幅度
    avg_gap = np.mean(gap_history)
    max_gap = np.max(gap_history)
    
    # 综合缓冲（取更保守的）
    buffer_1 = 2 * volatility_20d / (252**0.5)  # 2倍日波动
    buffer_2 = avg_gap * 1.5                     # 1.5倍平均跳空
    buffer_3 = max_gap * 0.8                     # 0.8倍最大跳空
    
    return max(buffer_1, buffer_2, buffer_3)
```

- 收盘前："临近收盘，保证金限制已收紧至60%"
- 隔夜："隔夜持仓需额外预留X%缓冲"

#### 3.5.1 双指标验证

```python
def volatility_filter(symbol, current_price, atr_14, ewma_vol_20):
    """
    波动率异常过滤（双指标验证）
    
    指标1: ATR(14) 相对历史均值
    指标2: EWMA波动率(20) 相对历史分位数
    
    触发条件: 任一指标超过阈值
    """
    # ATR检查
    atr_history = get_atr_history(symbol, 60)  # 60日历史
    atr_mean = np.mean(atr_history)
    atr_std = np.std(atr_history)
    atr_zscore = (atr_14 - atr_mean) / atr_std
    
    # EWMA波动率检查
    vol_percentile = calculate_percentile(ewma_vol_20, symbol, 252)
    
    # 触发条件
    if atr_zscore > 3.0:           # 3倍标准差
        return "BLOCK", f"ATR异常: Z-score={atr_zscore:.2f}"
    
    if vol_percentile > 95:         # 95%分位数
        return "BLOCK", f"波动率异常: 历史分位数={vol_percentile:.1f}%"
    
    if atr_zscore > 2.5 and vol_percentile > 90:  # 双重预警
        return "WARN", f"波动率预警: ATR Z={atr_zscore:.2f}, 分位数={vol_percentile:.1f}%"
    
    return "PASS", "波动率正常"
```

**触发后果：**
- BLOCK：禁止开仓，已有持仓可平仓
- WARN：允许开仓，但提示"当前波动率较高，建议降低仓位"

```python
def liquidity_check(symbol, order_volume, order_side):
    """
    流动性检查
    
    指标1: 订单量 / 20日日均成交量 < 5%
    指标2: 订单量 < 盘口深度（买一/卖一挂单量）
    """
    # 成交量检查
    avg_volume_20d = get_avg_volume(symbol, 20)
    volume_ratio = order_volume / avg_volume_20d
    
    if volume_ratio > 0.05:         # 超过5%

    depth = get_orderbook_depth(symbol, order_side)
    if order_volume > depth * 0.3:  # 超过盘口深度30%
        return "WARN", f"订单量接近盘口深度，建议拆单"
    
    return "PASS", "流动性充足"
```

### R9：资金充足性检查（★细化修订）

**核心变化：** 细化冻结资金计算，增加预冻结机制

```python
def capital_adequacy_check(account, order, existing_orders):
    """
    资金充足性检查（细化版）
    
    计算:
        可用资金 = 总权益 - 已占用保证金 - 冻结资金 - 预冻结
    """
    total_equity = account.balance
    used_margin = account.margin
    
    # 冻结资金（已挂单未成交）
    frozen_margin = sum(
        calc_margin(o) for o in existing_orders 
        if o.status == "PENDING"
    )
    
    # 预冻结（本笔订单）
    pre_frozen = calc_margin(order)
    
    # 安全缓冲（5%）
    safety_buffer = total_equity * 0.05
    
    available = total_equity - used_margin - frozen_margin - safety_buffer
    
    if pre_frozen > available:
        return "BLOCK", f"可用资金{available:.0f}不足，需{pre_frozen:.0f}"
    
    return "PASS", "资金充足"
```

```python
    """
    宏观熔断恢复机制
    
    防止在临界点附近频繁触发/恢复
    """
    hysteresis = 5  # 滞后区间

        return current_score < 70 - hysteresis  # 需回落至65分才恢复
```

### R11：处置效应监控（★新增）

**理论基础：** Shefrin & Statman (1985) 处置效应
> 投资者倾向于过早卖出盈利头寸、过晚卖出亏损头寸

#### 3.11.1 监控指标

```python
def disposition_effect_monitor(position, trade_history):
    """
    处置效应监控
    
    触发条件:
        1. 盈利头寸回撤超过50%未止盈
        2. 亏损头寸持仓时间 > 2倍平均盈利持仓时间
        3. 盈亏比 < 1.0（平均盈利 < 平均亏损）
    """
    alerts = []
    
    # 指标1: 盈利回撤监控
    if position.unrealized_pnl > 0:
        entry_price = position.entry_price
        current_price = position.current_price
        max_price = position.max_price_since_entry
        
        if max_price > entry_price:
            drawdown = (max_price - current_price) / (max_price - entry_price)
            if drawdown > 0.50:  # 盈利回撤超50%
                alerts.append({
                    "type": "PROFIT_DRAWDOWN",
                    "message": f"盈利回撤{drawdown*100:.0f}%，建议止盈",
                    "severity": "HIGH"
                })
    
    # 指标2: 亏损持仓时间监控
    if position.unrealized_pnl < 0:
        avg_profit_holding_time = calculate_avg_holding_time(
            trade_history, profitable_only=True
        )
        current_holding_time = position.holding_time
        
        if current_holding_time > 2 * avg_profit_holding_time:
            alerts.append({
                "type": "LONG_LOSS_HOLDING",
                "message": f"亏损持仓时间{current_holding_time}天，远超平均盈利持仓{avg_profit_holding_time}天",
                "severity": "MEDIUM"
            })
    
    # 指标3: 盈亏比监控
    avg_profit = calculate_avg_trade(trade_history, profitable_only=True)
    avg_loss = calculate_avg_trade(trade_history, losing_only=True)
    
    if avg_loss > 0 and avg_profit / avg_loss < 1.0:
        alerts.append({
            "type": "POOR_RR_RATIO",
```

### 5.1 压力测试模块

```python
def stress_test(portfolio, nav, scenarios=None):
    """
    组合压力测试
    
    参数:
        portfolio: 当前持仓组合
        nav: 账户净值
        scenarios: 自定义压力场景（None则使用标准场景）
    
    返回:
        results: 各场景下的盈亏和剩余保证金
    """
    if scenarios is None:
        scenarios = STANDARD_SCENARIOS
    
    results = {}
    for scenario_name, shocks in scenarios.items():
        # 应用价格冲击
        scenario_pnl = 0
        for position in portfolio.positions:
            symbol = position.symbol
            cluster = get_symbol_cluster(symbol)
            
            # 获取该品种/聚类的冲击幅度
            shock = shocks.get(symbol, shocks.get(cluster, 0))
            
            # 计算盈亏
            price_change = position.entry_price * shock
            pnl = price_change * position.volume * get_contract_size(symbol)
            scenario_pnl += pnl
        
        # 计算剩余净值和保证金
        remaining_nav = nav + scenario_pnl
        margin_ratio = portfolio.total_margin / remaining_nav
        
        results[scenario_name] = {
            "pnl": scenario_pnl,
            "remaining_nav": remaining_nav,
            "drawdown": -scenario_pnl / nav,
            "margin_ratio": margin_ratio,
            "survive": margin_ratio < 1.0 and remaining_nav > 0
        }
    
    return results
```

## 六、凯利公式仓位计算器（★新增）

### 6.1 计算器工具

```python
def kelly_position_sizer(win_rate, profit_loss_ratio, nav, symbol_volatility):
    """
    凯利公式仓位计算器
    
    参数:
        win_rate: 策略胜率（如0.55）
        profit_loss_ratio: 盈亏比（如1.5）
        nav: 账户净值
        symbol_volatility: 品种波动率
    
    返回:
        建议仓位（保证金占用）
    """
    # 凯利公式最优比例
    kelly_f = (win_rate * profit_loss_ratio - (1 - win_rate)) / profit_loss_ratio
    
    # 半凯利（更保守）
    half_kelly = kelly_f / 2
    
    # 波动率调整
    if symbol_volatility > 0.25:
        vol_adjust = 0.7
    elif symbol_volatility > 0.15:
        vol_adjust = 0.85
    else:
        vol_adjust = 1.0
    
    # 最终建议仓位
    suggested_position = nav * half_kelly * vol_adjust
    
    return {
        "kelly_full": nav * kelly_f,
        "kelly_half": nav * half_kelly,
        "vol_adjusted": suggested_position,
        "max_position": min(suggested_position, nav * 0.20)  # 不超过20%
    }
```

```
- 盈亏比：1.5
- 账户净值：100万
- 品种波动率：20%（铜）

凯利计算：
- 全凯利：16.7% = 16.7万
- 半凯利：8.3% = 8.3万

- 凯利建议更保守：7.1万
```
