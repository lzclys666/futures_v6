## 一、什么是风控？为什么需要？

### 1.1 通俗解释

风控 = **防止亏光钱的保险系统**

就像开车需要安全带、刹车、红绿灯一样，交易系统也需要多层保护：
- **安全带** → 止损规则（亏到一定幅度自动停车）
- **刹车** → 仓位限制（速度不能太快）
- **红绿灯** → 交易时间检查（不能闯红灯）

### 1.2 没有风控会怎样？

| 场景 | 后果 |
|------|------|
| 重仓押注单一品种 | 一次黑天鹅事件，账户归零 |
| 亏损后不断加仓摊平 | 越亏越多，陷入死循环 |
| 夜盘持仓无保护 | 隔夜跳空，爆仓穿仓 |
| 连续亏损情绪失控 | 报复性交易，加速亏损 |

**真实案例：** 2020年原油宝事件，无涨跌停保护 + 无仓位限制，投资者倒欠银行。

## 三、逐条规则详解

### R1：单品种最大持仓

**通俗说法：** 一个品种最多投多少钱？

**规则内容：**
- 每个品种最多占用账户资金的 **20%-30%**
- 高波动品种（如橡胶、焦煤）降至 **20%**
- 低波动品种（如国债期货）可放宽至 **30%**

**计算示例：**
```
账户资金：100万元
交易品种：黄金（AU）
黄金价格：500元/克
合约规格：1000克/手
保证金比例：10%

1手黄金保证金 = 500 × 1000 × 10% = 50,000元
最大持仓资金 = 100万 × 30% = 30万元
最大手数 = 30万 ÷ 5万 = 6手
```

**触发后果：** 想开第7手？系统拒绝，并提示"已超过单品种持仓上限"

**为什么要这样？** 防止单一品种暴跌导致账户重创

---

### R2：单日最大亏损

**通俗说法：** 今天最多亏多少钱？

**规则内容：**
- 单日亏损达到账户净值的 **2%**，禁止再开仓
- 平仓不受限制（可以止损离场）

**计算示例：**
```
账户净值：100万元
单日最大亏损 = 100万 × 2% = 2万元

情况1：上午亏了1.5万 → 还能开仓
情况2：下午累计亏2.1万 → 禁止开仓，只能平仓
```

**触发后果：** 系统提示"今日已亏损XX元，超过2%限制，请明日再交易"

**为什么要这样？** 防止情绪化交易，连亏时强制冷静

**注意：** 这个规则每天凌晨自动重置

---

### R3：涨跌停限制

**通俗说法：** 涨停不买，跌停不卖

**规则内容：**
- 买入开仓：委托价格 ≥ 涨停价 → 拒绝
- 卖出开仓：委托价格 ≤ 跌停价 → 拒绝

**示例：**
```
某合约：
- 昨日结算价：10000元
- 涨跌停幅度：±8%
- 涨停价：10800元
- 跌停价：9200元

如果你试图：
- 10800元买入开仓 → 拒绝（追涨停风险极高）
- 9200元卖出开仓 → 拒绝（抄底摸顶危险）
```

**触发后果：** 订单被拒绝，提示"价格触及涨跌停限制"

**为什么要这样？** 防止极端行情下追涨杀跌，避免流动性陷阱

---

### R4：总保证金占用上限

**通俗说法：** 不能把所有钱都押上

**规则内容：**
- 所有持仓占用的保证金总和 ≤ 账户权益的 **80%**
- 保留20%作为安全垫，应对波动

**计算示例：**
```
账户权益：100万元
最大保证金占用 = 100万 × 80% = 80万元

当前持仓：
- 黄金3手：15万保证金
- 铜5手：20万保证金
- 橡胶10手：25万保证金
合计：60万

还能开多少？80万 - 60万 = 20万保证金额度
```

**触发后果：** 提示"保证金占用已达80%，请先平仓再开仓"

**为什么要这样？** 防止满仓操作，无余地应对反向波动

---

### R5：波动率异常过滤

**通俗说法：** 市场疯了，先躲躲

**规则内容：**
- 当品种波动率超过近期平均的 **3倍**，禁止开仓
- 波动率用ATR（平均真实波幅）衡量

**示例：**
```
黄金近期平均波动：每天±10元
今天突然波动±35元 → 3.5倍 → 触发过滤

系统判断："市场异常，暂停交易"
```

**触发后果：** 禁止新开仓，已有持仓可平仓

**为什么要这样？** 异常波动往往伴随重大消息，此时交易风险极高

---

### R6：流动性检查

**通俗说法：** 太冷清的场子别去

**规则内容：**
- 下单量 > 该品种日均成交量的 **5%** → 拒绝
- 防止大单冲击市场，造成滑点损失

**示例：**
```
某小品种日均成交：1000手
你想下单：100手 → 占10% → 拒绝

建议：分多笔小单，或选择流动性更好的时段
```

**触发后果：** 提示"订单量超过流动性限制，请拆单或减少手数"

**为什么要这样？** 流动性差的品种，大单可能无法成交或价格严重偏离

---

### R7：连续亏损次数限制

**通俗说法：** 连输5把，歇一歇

**规则内容：**
- 连续亏损达到 **5次**，禁止开仓
- 平仓不受限制
- 次日自动重置

**示例：**
```
交易记录：
1. 做多黄金，止损离场 → 亏
2. 做空铜，止损离场 → 亏
3. 做多橡胶，止损离场 → 亏
4. 做空黄金，止损离场 → 亏
5. 做多铜，止损离场 → 亏

第6次想开仓 → 系统拒绝："连续亏损5次，请冷静后再交易"
```

**触发后果：** 强制暂停交易，防止情绪化报复性操作

**为什么要这样？** 连续亏损往往意味着市场状态变化或策略失效，需要暂停反思

---

### R8：交易时间检查

**通俗说法：** 非交易时间别下单

**规则内容：**
- 每个品种有固定的交易时段
- 非交易时段下单 → 拒绝

**常见品种交易时间：**

| 品种 | 日盘 | 夜盘 |
|------|------|------|
| 黄金/白银 | 09:00-15:00 | 21:00-02:30 |
| 铜/铝/锌 | 09:00-15:00 | 21:00-01:00 |
| 螺纹钢/橡胶 | 09:00-15:00 | 21:00-23:00 |
| 股指期货 | 09:30-15:00 | 无 |

**特殊时段保护：**
- 集合竞价（08:55-09:00, 20:55-21:00）：禁止开仓

**触发后果：** 提示"当前不在交易时段内"

**为什么要这样？** 非交易时段无流动性，价格可能失真

---

### R9：资金充足性检查

**通俗说法：** 没钱别下单

**规则内容：**
- 下单前检查：可用资金 ≥ 所需保证金
- 考虑已占用保证金和冻结资金

**计算示例：**
```
账户总权益：100万
已占用保证金：60万
冻结资金（挂单）：5万
可用资金 = 100万 - 60万 - 5万 = 35万

想开1手黄金（需5万保证金）→ 可以
想开10手黄金（需50万保证金）→ 拒绝
```

**触发后果：** 提示"可用资金不足，需XX元"

**为什么要这样？** 防止保证金不足导致强平

---

### R10：宏观熔断

**通俗说法：** 系统级警报，全员撤退

**规则内容：**
- 宏观评分极端看空（< 30分）：禁止做多
- 宏观评分极端看多（> 70分）：禁止做空
- 相当于市场的"红色警报"

**评分含义：**
```
0-30分：极端看空 → 禁止做多（只能做空或观望）
30-40分：看空 → 建议做空
40-60分：中性 → 双向交易
60-70分：看多 → 建议做多
70-100分：极端看多 → 禁止做空（只能做多或观望）
```

**触发后果：** 提示"宏观熔断触发，当前禁止做多/做空"

**为什么要这样？** 极端行情下，逆势交易风险极高，系统强制保护

## 五、常见问题 FAQ

### Q1：规则触发后，我能平仓吗？

**A：** 大部分规则只限制**开仓**，不限制**平仓**。但以下情况例外：
- 涨跌停限制：涨停时不能平多（没有买盘），跌停时不能平空（没有卖盘）
- 流动性检查：极端情况下可能无法成交

### Q2：连续亏损5次后，我只能干看着？

**A：** 是的，但你可以：
1. 检查策略是否需要调整
2. 分析市场状态是否变化
3. 等待次日自动重置
4. 联系风控员申请手动解除（需审批）

### Q3：为什么我的订单被拒绝了，但我不清楚原因？

**A：** 查看系统提示信息，格式如下：
```
[风控拦截] 规则名：具体原因
例如：
[风控拦截] 单日最大亏损：今日已亏损15000元，超过限制10000元
```

### Q4：我能调整规则阈值吗？

**A：** 部分可以，部分不行：

| 可调整 | 不可调整 |
|--------|---------|
| 单品种持仓比例（20%-30%） | 涨跌停限制（交易所规定） |
| 单日亏损比例（1%-3%） | 交易时间（交易所规定） |
| 连续亏损次数（3-7次） | 保证金比例（交易所规定） |

**调整方式：** 联系系统管理员，通过配置文件修改

### Q5：规则会不会太严格，错过机会？

**A：** 风控的本质是"保命"而非"赚钱"。统计表明：
- 严格风控的账户，年化波动率降低40%
- 极端回撤减少60%
- 长期存活率提高3倍

**记住：** 活着才有输出，爆仓一次就归零。

## 八、紧急处理流程

### 8.1 系统故障

**现象：** 风控系统无响应，所有订单被拦截或全部通过

**处理步骤：**
1. 立即停止所有策略
2. 检查风控服务状态：`python scripts/check_risk_service.py`
3. 手动模式：联系风控员，人工审核每笔订单
4. 修复后重启，验证规则生效

### 8.2 极端行情

**现象：** 多品种同时触发熔断，无法开仓

**处理步骤：**
1. 确认是否为系统性风险（如战争、金融危机）
2. 已有持仓：评估是否强行平仓
3. 暂停交易，等待市场稳定
4. 风控员可临时调整熔断阈值（需记录审批）

### 8.3 误拦截申诉

**现象：** 确认规则触发不合理

**处理流程：**
```
交易员提交申诉 → 风控员审核 → 管理员审批 → 临时解除/调整规则
     ↑___________________________________________↓
                    （24小时内反馈）
```
